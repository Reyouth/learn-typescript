    什么是修饰符

    # 什么是修饰符

    限定类型以及类型成员的符号

    # Vue 修饰符

    修饰符修饰了DOM时间细节，可以让开发者专注于程序逻辑

    # Vue修饰符种类

    • 表单修饰符
    • 事件修饰符
    • 鼠标按键修饰符
    • 键值修饰符
    • v-bind修饰符

    # 表单修饰符

    表单里我们使用input标签跟v-model最多。 表单修饰符有：

    • lazy
    • trim
    • number

    # lazy

    再填完消息内容，光标离开标签的时候才会进行值同步，即懒同步，或者说触发change后才同步

    <input type="text" v-model.lazy="value"><p>{{value}}</p>

    # trim

    自动过滤输入内容的首空格，中间空格不过滤

    <input type="text" v-model.trim="value"><p>{{value}}</p>

    # number

    自动将输入内容转化为数值类型，不能被解析的话就变回原值

    <input type="text" v-model.number="value"><p>{{value}}</p>

    # 事件修饰符

    • stop
    • prevent
    • self
    • once
    • capture
    • passive
    • native

    # stop

    阻止事件冒泡，相当于event,stopPropagtion

    <div @click="shout(2)">
    <button @click.stop="shout(1)"></button>
</div>
<!-- 只输出1 -->

    # prevent

    阻止了事件的默认行为，相当于event.preventDefault

    <form v-on:submit.prevent="onSubmit"></form>

    # self

    只有event.target是自己触发的时候才处理

    <div @click.self="doThat">...</div>

    修饰符的顺序细节会影响语义 比如@click.prevent.self会阻止所有的点击 但是@click.self.prevent仅阻止自身点击

    # once

    仅触发一次，第二次不会触发

    # capture

    与阻止冒泡相反，事件触发会从包含这个元素的顶层开始往下触发，兄弟元素触发是先触发自己然后兄弟按顺序触发

    <div @click.capture="shout(1)">
    obj1
    <div @click.capture="shout(2)">
        obj2
        <div  @click.capture="shout(3)">
obj3
        </div>
        <div  @click.capture="shout(4)">
obj4
        </div>
    </div>
</div>
<!-- 1 2 4 3  -->

    # passive

    常用于移动端 当我们在移动端监听元素滚动事件，会一直触发onscroll，事件堆积会使得网页卡顿。使用passive相当于给onscroll加一个lazy，有效缓解卡顿。

    <!-- 滚动事件行为会立刻触发 -->
<!-- 不会等待'onScroll' -->
<!-- 包含了 event.preventDefault()情况 -->
<div @scroll.passive="onScroll">...</div>

    # native

    使得子组件像html内置标签一样只监听根元素的原生事件，即“退化”。否则组件只会使用v-on(@)监听自定义事件，可以用来父组件给子组件定义事件并添加，而不是在子组件中添加然后父组件调用

    所以修饰native会让普通html标签失效

    # 鼠标按钮修饰符

    -left -right -middle

    <button @click.left></button>

    # 键盘按键修饰符

    keyCode键位上区分两种

    • 普通键（tab enter delete esc）
    • 系统修饰键（ctrl shift）

    键盘事件分两类，按下跟松开

    • onkeydown
    • onkeyup

    也可以在全局设置键盘码别名

    Vue.config.keyCodes.f2=113

    # v-bind修饰符

    监听修饰符服务于属性操作

    • sync
    • prop
    • camel

    # sync

    对props双向绑定

    <!-- 父组件 -->
<comp :myMessage.sync="bar"></comp>
<!-- 子组件 -->
<script>
this.$emit("update:myMessage",params)
</script>

    这个语法糖相当于

    <!-- 父组件 -->
<comp :myMessage="bar" @update:myMessage="func"></comp>
<script>
    func(e){
        this.bar=e
    }
    </script>
<!-- 子组件 -->
<script>
this.$emit("update:myMessage",params)
</script>

    使用sync时，必须格式要正确，子组件emit参数名必带update:value，value名称要跟props声明的一致 sync 声明不能使用表达式

    # props

    设置自定义标签属性，避免暴露数据，防止污染

    # camel

    将命名转化为驼峰命名法

    # 总结应用场景

    • .stop 阻止事件冒泡
    • .native 绑定原生事件
    • .once 仅执行一次的事件
    • .self 将事件仅绑定自身，相当于阻止事件冒泡，区别是只顾自己，不一定父级不冒泡
    • .prevent 阻止默认事件
    • .capture 用于事件捕获
    • .keyCode 特定按键触发
    • right 鼠标右键
