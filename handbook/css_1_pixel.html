    CSS的一像素问题

    # CSS的一像素问题

    # 问题由来 iPhone4

    border 1px iPhone4使用的视网膜屏幕中引用了把2x2像素当作一个物理像素来用的概念，使得UI显示更加清晰。其中这个2x2像素称为逻辑像素，即像素比为2的屏幕，二倍屏的诞生，导致了CSS中原生定义的1px边框会在二倍屏中占用了2个逻辑像素的宽度，这是开发者不愿意看到的。

    # 解决方案

    • transform
    • 媒体查询（only in iOS8+）
    • box-shadow

    # transform:scale(0.5)

    优点：灵活，推荐 缺点：使用到伪元素，若已有伪元素则需要多嵌套一个无用元素

    1. 设置height: 1px，根据媒体查询结合transform缩放为相应尺寸。
    div {
    height:1px;
    background:#000;
    -webkit-transform: scaleY(0.5);
    -webkit-transform-origin:0 0;
    overflow: hidden;
}
    1. 用::after和::befor,设置border-bottom：1px solid #000,然后在缩放-webkit-transform: scaleY(0.5);可以实现两根边线的需求
    div::after{
    content:'';width:100%;
    border-bottom:1px solid #000;
    transform: scaleY(0.5);
}
    1. 用::after设置border：1px solid #000; width:200%; height:200%,然后再缩放scaleY(0.5); 优点可以实现圆角
    .div::after {
    content: '';
    width: 200%;
    height: 200%;
    position: absolute;
    top: 0;
    left: 0;
    border: 1px solid #bfbfbf;
    border-radius: 4px;
    -webkit-transform: scale(0.5,0.5);
    transform: scale(0.5,0.5);
    -webkit-transform-origin: top left;
}

    # 媒体查询

    .border { border: 1px solid #999 }
@media screen and (-webkit-min-device-pixel-ratio: 2) {//2倍
    .border { border: 0.5px solid #999 }
}
@media screen and (-webkit-min-device-pixel-ratio: 3) {//3倍
    .border { border: 0.333333px solid #999 }
}

    <body><div id="main" style="border: 1px solid #000000;"></div></body>
<script type="text/javascript">
    if (window.devicePixelRatio && devicePixelRatio >= 2) {
        var main = document.getElementById('main');
        main.style.border = '.5px solid #000000';
    }
</script>

    # box-shadow

    利用阴影也可以实现

    div {
    -webkit-box-shadow: 0 1px 1px -1px rgba(0, 0, 0, 0.5);
}

    box-shadow: h-shadow v-shadow [blur] [spread] [color] [inset]

    参数分别表示: 水平阴影位置，垂直阴影位置，模糊距离， 阴影尺寸，阴影颜色，将外部阴影改为内部阴影，后四个可选。该例中为何将阴影尺寸设置为负数？设置成-1px 是为了让阴影尺寸稍小于div元素尺寸，这样左右两边的阴影就不会暴露出来，实现只有底部一边有阴影的效果。从而实现分割线效果

    参考链接：https://www.jianshu.com/p/31f8907637a6
